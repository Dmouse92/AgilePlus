# Rich Journey Embeds

## Problem
User-facing docs lack a canonical journey-capture stub pattern. The hwLedger stash has the closest precedent.

## Target Users
Docs authors, implementers, reviewers, and autograder maintainers.

## Functional Requirements
- Every docs page that walks a user story MUST include a journey stub:
  - `![journey:<slug>](capture/<slug>.gif)`
  - or `<!-- EMBED journey:<slug> capture/<slug>.gif -->`
- Stub presence MUST be enforced by docs lint.
- Real captures MUST be generated by a CI step once available.
- The stub path or HTML comment MUST appear in the spec-docs-tests-code trace.

## Acceptance Criteria
- Journey-stub lint is part of the eco-021 autograder.
- Missing stubs fail docs lint.
- Trace output links specs, docs, tests, code, and embed stubs.

## Constraints
- eco-018 spec-first.
- eco-019 worktree-isolation.
- eco-021 gates.
