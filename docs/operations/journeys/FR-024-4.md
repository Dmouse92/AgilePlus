# Journey: CI gate on every PR

> **Status: stub** — eco-024 FR-6.

## User story
As a **PR author**, I push a PR and the `agileplus-traceability.yml` workflow runs the validator; merges are blocked if any FR is missing a trace.

## Steps
1. Open PR.
2. CI runs the validator.
3. Required check appears in PR status.
4. PR is mergeable only on green.

## Acceptance
- A PR that drops a trace entry cannot be merged.
